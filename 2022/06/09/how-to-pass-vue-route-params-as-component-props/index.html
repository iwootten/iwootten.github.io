How to Pass Vue Route Params as Component Props

  • vue
  • js

If youâ€™re using vues router to link to dynamic URLs you may get stuck wondering why your props on router-links arenâ€™t being sent to your components. Vueâ€™s router doesnâ€™t support sending props directly. So you canâ€™t do something like this:

<router-link
  class="..."
  :to="{
     path: `/user/edit/`,
     id=id
   }"
   >Edit
</router-link>

Here the id prop will never get passed in routers default configuration. Instead you need to configure each of your routes to pass the route params as props like so:

<router-link
  class="..."
  :to="{
     path: `/user/edit/${id}`
   }"
   >Edit
</router-link>
const UserEdit = {
  props: ['id'],
  template: '<div>Edit User {{ id }}</div>'
}

const routes = [{ 
	path: '/user/edit/:id', 
  component: UserEdit, 
  props: true 
}]

This way, the id comes through as a prop to our component without an issue. Router allows this to avoid tight coupling between components and route params.
